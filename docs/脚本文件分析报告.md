## 📈 为什么会产生这么多脚本？

### 1. **开发迭代**
- 多个版本的启动脚本（start.sh, start-server.sh, start_server.sh）
- 功能演进导致脚本重复

### 2. **问题修复**
- Session锁定问题 → `fix_session_locks.sh`
- 阶段性修复 → `apply_stage1_fixes.sh`
- 文件整理 → `move_test_files.sh`, `organize_files.sh`

### 3. **功能扩展**
- 服务器监控 → `monitor_server.sh`
- 环境检查 → `check-env.sh`
- 任务状态查询 → `check_task_status.sh`

### 4. **维护需求**
- 备份恢复 → `create_backup.sh`, `restore_file.sh`
- 清理工具 → `cleanup_backups.sh`
- Cron设置 → `setup-cron.sh`
