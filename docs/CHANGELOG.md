# GEOFlow 更新日志

该文档记录公开仓库可见功能的持续更新。后续每次推送到 GitHub 时，同步更新本文件和英文版 `CHANGELOG_en.md`。

## 2026-07-17

### v2.1.1（发布准备）

- 加固前台结构化数据输出：
  - 所有主题的 JSON-LD 统一使用 Laravel `Js::encode` 生成，阻断 `</script>` 等可执行上下文注入，同时保留合法 Schema 数据。
- 收紧受管图片和 API 幂等边界：
  - 图片上传改用内容寻址文件名和受管目录校验，新增 `images.managed_path_hash`、`managed_image_paths` 注册表、状态与 fencing 信息，避免外部路径、符号链接和并发删除越界。
  - API 幂等记录新增 durable state、owner lease 与 fingerprint version，旧记录和过期的 `in_progress` 记录进入显式人工处理路径。
  - 物理图片删除保持默认关闭，升级必须停机排空旧进程、确认迁移、完成 `managed_path_hash` 回填和 readiness 检查后再开启。
- 统一出站请求与敏感后台权限：
  - 分发、URL 导入、主题参考抓取、AI、更新检查与归档下载统一经过出站安全网关，覆盖 URL 规范化、DNS 全量校验、IP 固定、重定向控制、响应体上限和错误脱敏，修复 SSRF 绕过路径。
  - 分发、URL 导入、主题和复刻相关敏感管理路由统一要求超级管理员权限。
- 隔离生成主题代码：
  - 关闭可写入 Blade/CSS 的在线主题编辑入口和路由；主题复刻预览使用受信任的确定性页面，不编译生成的 Blade 代码，并以 sandbox CSP 禁止脚本与外部资源。
  - 主题复刻发布改为 package-only 审查包下载，不再把生成文件写入运行中的主题目录。
- 新增只读安全审计：
  - `php artisan geoflow:security-audit` 提供人工可读结果，`--json` 提供稳定 schema；任意 finding 或审计无法完成时返回退出码 `1`。
  - 审计覆盖安全迁移、受管图片注册表、删除门禁、API 幂等状态、旧图片路径输入和私网出站例外，不发起网络请求、DNS 查询或数据修复。
- 更新依赖安全基线：
  - 升级 Laravel、Guzzle、PSR-7 和 Symfony 安全补丁，锁文件依赖审计无已知公告。
  - 项目最低 PHP 版本统一为 8.3，与当前 `laravel/ai` 要求一致；Docker 默认继续使用 PHP 8.4。

## 2026-06-26

### v2.1.0

- 新增企业知识库草稿生成工作流：
  - 后台新增“企业知识库”入口，支持创建项目、上传或粘贴多份来源资料、查看来源解析结果、触发 AI 草稿生成和查看历史修订。
  - 新增项目、来源、修订三类数据表和队列任务，草稿生成可在后台异步执行，避免长资料阻塞管理页面。
  - 来源解析支持文本、Markdown、HTML、CSV、JSON、XML、PDF、Word 和 Excel 等常见资料格式，并统一沉淀为可追踪来源。
  - AI 清洗和结构化整理改为更重视原文覆盖，长资料会按模块拆分处理，并把遗漏的来源事实回填到对应章节，减少上传资料被过度简化的问题。
  - 草稿结果保留模块、目标读者、内容形态、待人工确认项和来源摘要，便于继续人工审核与发布。
- 优化知识库与素材管理：
  - 知识库详情页新增 Markdown 编辑器，支持在后台直接查看和调整知识内容。
  - 知识库详情页支持重新提交语义切片，并能从详情页安全返回。
  - 素材管理页增加企业知识库构建入口，让普通知识库、素材库和企业知识库草稿工作流更容易串联。
- 增强主题、模板和前台输出：
  - 新增站点主题在线编辑能力，可在后台编辑当前主题模板并保留测试覆盖。该在线主题编辑能力已在 v2.1.1 中关闭，当前安全流程使用隔离预览和 package-only 审查包。
  - 站点设置新增首页模块配置和自定义样式能力，目标站点包同步支持更丰富的首页结构。
  - 新增 APIHot 推荐类前台主题，补充首页、分类、归档、文章页和配套静态资源。
  - 统一前台 SEO metadata 输出，主题页面共享 SEO head 逻辑，减少不同主题之间标题、Open Graph 和结构化数据不一致的问题。
- 改善分发管理和目标站点同步：
  - 任务新增分发策略，可按任务配置本地、渠道和本地加渠道的分发方式。
  - 文章列表增强分发状态可视化，方便直接查看远端副本、同步状态和失败信息。
  - 分发管理页支持批量选择启用中的 GEOFlow Agent 渠道并同步目标站点设置。
  - 目标站点包补充统一 SEO metadata 契约，前台和远端静态页面的搜索输出更一致。
- 提升部署与运行稳定性：
  - 新增安装状态标记与默认数据写入保护，避免已有部署在重启、迁移或升级时重复写入演示数据。
  - 默认管理员初始化支持邮箱配置，并补充生产入口脚本、Redis session、Docker 镜像、网络、权限和反向代理相关改进。
  - 后台路径和 Reverb 鉴权路径支持子目录部署，降低 `APP_URL` 带路径前缀时的静态资源和 WebSocket 配置问题。
- 扩展测试覆盖：
  - 新增企业知识库、主题编辑、站点设置、分发策略、目标站点同步、SEO metadata、安装保护和部署配置相关测试。
  - 本次发布前完整测试结果为 `479 passed`、`3179 assertions`。
