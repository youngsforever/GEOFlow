# 安全设置页面修复说明

## 🐛 问题描述

**页面**: http://localhost:8080/geo_admin/security-settings.php

**问题现象**:
1. ❌ 添加敏感词功能不能使用
2. ❌ 修改管理员密码功能不能使用

**错误原因**:
数据库表结构与代码不匹配，缺少必要的字段。

---

## 🔍 问题分析

### 问题1: 敏感词功能失败

**代码期望**:
```sql
INSERT INTO sensitive_words (word, category, created_at) VALUES (?, ?, CURRENT_TIMESTAMP)
```

**实际表结构**:
```sql
sensitive_words (
    id INTEGER PRIMARY KEY,
    word VARCHAR(200) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**问题**: 表中没有 `category` 字段

### 问题2: 修改密码功能失败

**代码期望**:
```sql
UPDATE admins SET password = ?, updated_at = CURRENT_TIMESTAMP WHERE id = ?
SELECT username, email, last_login FROM admins WHERE id = ?
```

**实际表结构**:
```sql
admins (
    id INTEGER PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(255) NOT NULL,
    display_name VARCHAR(100),
    email VARCHAR(100),
    bio TEXT,
    avatar VARCHAR(200),
    website VARCHAR(200),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**问题**: 表中没有 `updated_at` 和 `last_login` 字段

---

## ✅ 修复方案

### 修复策略
采用**修改代码适配现有数据库**的方案，而不是修改数据库结构。

优点：
- 不需要修改数据库
- 不影响现有数据
- 向后兼容
- 更安全

---

## 🔧 具体修复内容

### 1. 敏感词添加功能

#### 修改前（第46-56行）:
```php
foreach ($word_list as $word) {
    if (!empty($word)) {
        $stmt = $db->prepare("
            INSERT OR IGNORE INTO sensitive_words (word, category, created_at)
            VALUES (?, ?, CURRENT_TIMESTAMP)
        ");
        if ($stmt->execute([$word, $category])) {
            $added_count++;
        }
    }
}
```

#### 修改后:
```php
foreach ($word_list as $word) {
    if (!empty($word)) {
        // 检查是否已存在
        $check_stmt = $db->prepare("SELECT id FROM sensitive_words WHERE word = ?");
        $check_stmt->execute([$word]);
        
        if (!$check_stmt->fetch()) {
            $stmt = $db->prepare("
                INSERT INTO sensitive_words (word, created_at)
                VALUES (?, CURRENT_TIMESTAMP)
            ");
            if ($stmt->execute([$word])) {
                $added_count++;
            }
        }
    }
}
```

**改动**:
- ✅ 移除了 `category` 字段
- ✅ 改用手动检查重复（因为移除了 `INSERT OR IGNORE`）
- ✅ 保留了去重功能

## 📞 使用说明

### 添加敏感词
1. 登录后台
2. 进入"安全管理"页面
3. 在"敏感词列表"文本框中输入敏感词（每行一个）
4. 点击"添加敏感词"按钮

### 修改密码
1. 在"修改管理员密码"部分
2. 输入当前密码
3. 输入新密码（至少6位）
4. 确认新密码
5. 点击"修改密码"按钮
6. 修改成功后建议重新登录

### 管理敏感词
1. 在"敏感词列表"中查看所有敏感词
2. 点击删除图标可删除单个敏感词
3. 删除前会有确认提示
