# URL 智能采集模块 PRD

更新时间：2026-03-27

## 1. 目标

在后台“素材管理”中新增一个独立模块，允许管理员输入单个网页 URL，通过系统自动抓取、正文提取、AI 清洗与语义分析，一键生成并写入以下素材资产：

- AI 知识库
- 关键词库
- 标题库
- 图片库

目标不是做通用爬虫，而是做“单页高质量采集与结构化入库”。

## 3. 用户价值

当前素材管理依赖手工分别录入知识、关键词、标题和图片，效率低且内容来源割裂。

新增 URL 智能采集后，管理员只需提供一个高质量页面地址，即可完成：

- 页面正文抽取
- 知识库内容沉淀
- 语义关键词生成
- 标题库自动生成
- 正文主图采集
- 来源追溯与去重

这会把素材管理从“手工录入工具”升级成“内容资产加工入口”。

## 4. 核心原则

1. 只做单页采集，不先做整站爬虫
2. 先预览后入库，不允许无确认直接写库
3. 只抓正文主体内容，不抓导航、推荐区、广告、页脚
4. 只抓正文主体图片，不抓 logo、头像、装饰图
5. 所有入库结果必须保留来源 URL 和抓取时间
6. 必须有重复检测，避免同一内容多次污染素材库

## 5. 产品流程

### 5.1 基础流程

1. 管理员在 [materials.php](/Users/laoyao/AI Coding/01-Projects/Active/GEO官网系统/geo_admin/materials.php) 点击“URL 智能采集”
2. 打开 [url-import.php](/Users/laoyao/AI Coding/01-Projects/Active/GEO官网系统/geo_admin/url-import.php)
3. 输入 URL 和采集参数

7. 展示预览页
8. 管理员勾选要入库的结果并确认
9. 系统写入对应素材库
10. 记录采集历史与来源映射

目标归属：

## 7. 技术流程

### 7.1 抓取层

目标：

- 获取页面完整 HTML
- 识别文章主区域
- 提取正文、标题、meta、图片、表格、列表

建议实现：

- 首选 `cURL` 抓取 HTML
- 支持自定义 User-Agent
- 跟随 301/302
- 设置超时和最大响应体大小
- 对常见编码进行统一转码为 UTF-8

提取内容：

- `title`
- `meta_description`
- `canonical_url`
- `h1`
- 正文段落
- 有序列表 / 无序列表
- 表格
- 图片 URL、alt、所在段落

### 7.2 正文抽取层

目标：

- 去掉页头、页脚、侧栏、推荐区、广告、版权声明等模板噪声

建议策略：

- 基于 DOM 密度与文本密度识别正文容器
- 优先保留：
  - 主标题
  - 段落
  - 小标题
  - 表格
  - FAQ
  - 步骤列表
- 排除：
  - 导航菜单
  - 面包屑
  - “猜你喜欢”
  - 相关文章
  - 评论区
  - 页脚

### 7.3 AI 清洗层

目标：

- 将网页原文转换为适合素材库使用的高质量结构化内容

输出建议：

- `clean_content`
- `summary`
- `key_facts`
- `faq`
- `semantic_keywords`
- `generated_titles`
- `image_candidates`

清洗要求：

- 去广告和营销噪声
- 去重复段落
- 保留统计数字、引语、来源、术语
- 保持客观中立
- 保留结构化段落和标题层级

## 8. 入库规则

### 8.1 知识库

写入目标：

- [admin/knowledge-bases.php](/Users/laoyao/AI Coding/01-Projects/Active/GEO官网系统/geo_admin/knowledge-bases.php) 对应数据表

写入字段建议包含：

- 标题
- 清洗后正文
- 摘要
- 来源 URL
- 来源域名
- 抓取时间
- 来源页面标题
- 采集任务 ID

### 8.2 关键词库

生成原则：

- 基于页面主题 + AI 语义分析
- 优先保留高信息增益的主题词、术语词、实体词、查询词
- 区分核心关键词和扩展关键词

建议输出：

- 核心关键词
- 语义扩展词
- 查询重写词
- 长尾问题词

### 8.3 标题库

生成原则：

- 基于页面核心主题、提取关键词和语义意图自动生成
- 标题需去重
- 标题需可用于后续任务生成

建议分组：

- 信息型标题
- 对比型标题
- 指南型标题
- 问题型标题

### 8.4 图片库

只采集正文主体图。

图片入库前需检查：

- URL 是否有效
- 是否重复
- 是否为极小图标或装饰图
- 是否支持下载

写入字段建议：

- 原始图片 URL
- 本地保存路径
- alt 文本
- 来源 URL
- 来源页面标题
- 采集任务 ID

## 10. 去重策略

必须至少做四层判重：

1. URL 判重
2. 规范化 URL 判重
3. 页面标题 + 域名判重
4. 正文内容 hash 判重

处理策略：

- 默认提示“可能重复”
- 管理员可选择：
  - 取消导入
  - 继续导入为新版本
  - 覆盖旧内容

- 从网页正文中提取高价值知识内容

输出格式建议：

- 摘要

- 关键事实

### 11.2 关键词提示词

目标：

- 生成适合作为 GEO/SEO/内容策划的关键词集合

输出建议：

- 核心词
- 扩展词
- 长尾问句

### 11.3 标题生成提示词

目标：

- 基于页面内容和关键词生成可写作、可发布、可检索的标题

输出建议：

- 10-20 个标题
- 去重
- 按类型分组

## 12. 异常与失败处理

需处理的失败类型：

- URL 非法
- 页面无法访问
- 重定向过多
- HTML 抓取失败
- 正文提取失败
- AI 清洗失败
- 图片下载失败
- 入库失败

后台提示应明确到阶段：

- 抓取失败
- 解析失败
- AI 处理失败
- 入库失败

不要只提示“处理失败”。

## 13. 权限与安全

要求：

- 仅管理员可使用
- 限制单次抓取的响应体大小
- 限制图片下载数量和大小
- 禁止抓取内网地址与本地地址
- 禁止 `file://`、`ftp://` 等危险协议
- 对 URL 做协议、域名和 IP 校验

重点防止：

- SSRF
- 本地文件读取
- 恶意超大页面
- 重复采集刷库
