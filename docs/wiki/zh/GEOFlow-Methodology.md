# GEOFlow 方法论

GEOFlow 的方法论可以概括为一句话：

> 先把知识变成资产，再把资产变成任务，最后把任务变成稳定可分发的内容输出。

它背后有 5 个核心原则。

## 1. 知识优先于生成

如果知识库本身不真实、不稳定、不成体系，再强的自动化也只会放大噪音。  
所以在 GEOFlow 里，知识库建设不是附属能力，而是优先级最高的基础设施。

## 2. 任务优先于单篇写作

单篇生成只能解决局部效率问题。  
真正的内容工程，需要把输入、规则、素材、审核和发布组织成任务，让它们可调度、可追踪、可复用。

## 3. 工程化优先于灵感驱动

GEOFlow 更强调：

- 规则
- 素材
- 模板
- 队列
- 状态
- 审核

而不是依赖一次性的“灵感型创作”。这是它和普通写作工具的核心差异。

## 4. 前台展示是内容系统的一部分

前台不是最后补一个壳子。  
在 GEO 场景里，模板、信息结构、SEO 信息、结构化数据、广告位、栏目组织，本身就是内容系统的一部分。

## 5. 扩展性必须内建

GEOFlow 从一开始就不是只面向 Web 后台。  
Skill、CLI、API、模板包、预览机制，都是为了让它能逐步演进成更强的内容操作平台。

所以它的方法论不是“多生成一点”，而是：

- 更真实
- 更结构化
- 更可维护
- 更适配 AI 搜索
- 更适合长期运营
