# 什么是 GEOFlow

GEOFlow 是一套围绕 GEO 内容工程设计的开源系统。

它的核心不是“生成一篇文章”，而是把内容生产、内容管理、内容审核、内容分发和前台呈现组织成一套可以长期运行的系统能力。

它不是这些东西：

- 不是一个只会生成文案的 AI 写作器
- 不是一个只负责发布文章的博客引擎
- 不是一个为堆量和信息污染服务的批量页面工具

它更接近这些角色：

- 面向 GEO 场景的内容管理系统
- 面向 AI 搜索的知识资产操作系统
- 面向内容团队的任务化生产平台
- 面向站点运营的模板化分发系统

GEOFlow 的典型链路是：

1. 配置模型和提示词
2. 建设标题库、图片库、关键词库、知识库等素材资产
3. 创建任务并进入调度
4. 由队列和 Worker 执行生成
5. 进入草稿、审核、发布流程
6. 输出前台页面、SEO 信息和结构化数据

因此，GEOFlow 的价值并不只在“写内容更快”，而在于：

- 把内容生产从单次操作变成可重复的任务
- 把知识沉淀从分散文件变成系统资产
- 把前台展示从一次性页面变成可维护模板
- 把 AI 调用从单模型试验变成可管理的系统能力

如果你的目标是：

- 建官网内容频道
- 建 GEO 信源站
- 做内部 GEO 内容中台
- 做多站点、多栏目、多模板的内容协同

那么 GEOFlow 是合适的。

如果你的目标只是：

- 批量制造低质量页面
- 快速堆砌无法核验的内容
- 用自动化替代真实性和判断

那么这套系统不适合，也不应该这样使用。
