# 推荐采用路径

如果你是第一次使用 GEOFlow，最稳的方式不是一下子把所有功能都打开，而是按阶段采用。

## 阶段一：先明确目标

先回答三个问题：

- 你希望服务谁
- 你希望解决什么内容问题
- 你希望站点成为哪种类型的内容资产

## 阶段二：先建设知识库

先把真实资料、事实信息、案例、FAQ、术语、产品信息整理起来。  
这是后面所有自动化的基础。

## 阶段三：配置模型和提示词

这一步的目标不是追求“最强模型”，而是找到：

- 稳定的模型组合
- 合适的提示词结构
- 能和知识库配合的生成策略

## 阶段四：建立任务和审核流程

把高频内容场景任务化，让它们能进入统一的调度和审核流程。

## 阶段五：完成前台与模板

等内容和流程稳定后，再逐步优化前台模板、主题和展示风格。

## 阶段六：再做自动化扩展

最后再接入：

- CLI
- Skill
- API
- 多站点
- 自动化分发

这个顺序的核心原则是：

> 先做真实、再做稳定、最后做规模。

如果顺序反过来，自动化通常只会放大混乱。
