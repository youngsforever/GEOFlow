# Skill / CLI / API 生态

GEOFlow 的长期方向不是只依赖后台页面操作。

## Web 后台

适合：

- 日常管理
- 人工审核
- 配置模型、素材、任务和模板

## CLI

适合：

- 本地批量操作
- 自动化脚本
- 任务维护与系统巡检

## API

适合：

- 与外部系统集成
- 程序化接入
- 自动化编排

## Skill

适合：

- 把重复流程沉淀成可复用能力
- 让 GEOFlow 与更复杂的工作流协同
- 做面向任务的自动化操作，而不是单点脚本

当前已经公开的配套 skill 包括：

- `geoflow-cli-ops`
- `geoflow-template`

它们分别对应：

- 面向 CLI 的系统运维与内容操作
- 面向前台模板映射和复刻的预览优先流程

未来 Skill / CLI / API 的价值，在于让 GEOFlow 从“后台系统”逐步演进成“内容操作平台”。
