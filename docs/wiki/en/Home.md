# GEOFlow Wiki Home

Welcome to the GEOFlow Wiki.

GEOFlow is an open-source system for GEO content engineering, AI-search-ready publishing, and automated content operations. It is not just a CMS and not just an AI writing tool. It is closer to a content operating system that connects models, materials, tasks, review, publishing, frontend delivery, and Skill / CLI / API collaboration into one workflow.

This wiki is meant to answer six questions:

- What GEOFlow actually is
- Which teams and scenarios it fits
- What its methodology is
- What its content boundaries and operating principles are
- How it should be deployed and adopted
- Where it is heading next

If there is one line that best describes GEOFlow:

> It turns knowledge assets, AI content generation, review, publishing, and frontend presentation into an engineered operating workflow.

Its current strengths include:

- Multi-model integration
- Centralized material libraries
- Task scheduling, queueing, and worker execution
- Draft, review, and publish workflows
- Search- and AI-citation-ready frontend output
- Theme preview and activation
- Skill / CLI / API collaboration

The system is meant to amplify the value of real knowledge assets, not to amplify noise.
