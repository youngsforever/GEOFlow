## 3. 问题清单

### P1 高优先级

当前无阻断型 P1 缺陷留存。以下项目已完成修复并转入“已修复记录”。

### P2 中优先级

#### 3.1 AI 模型 API Key 仍缺少密钥轮换与环境密钥治理

涉及页面：

- `admin/ai-models.php`
- `ai_models` 表

问题说明：

- `ai_models.api_key` 现已改为加密存储
- 后台页面不再明文展示或回填真实密钥，编辑时留空可保留旧密钥
- 当前仍依赖应用侧 `APP_SECRET_KEY` 进行加解密，尚未建立密钥轮换机制

影响：

- 数据库泄露风险已明显下降
- 但如果应用密钥管理不当，仍会影响整体安全性

建议修复：

- 将 `APP_SECRET_KEY` 放入部署环境变量，不使用默认值
- 后续补充密钥轮换与重加密脚本
- 审计访问权限

### P3 低优先级

#### 3.8 后台存在大量并行版本页面，维护成本高

涉及页面：

- `*-simple.php`
- `*-new.php`
- `*-backup.php`
- `test-*.php`

问题说明：

- 同类功能存在多套页面
- 正式入口和测试入口边界不够清晰

影响：

- 新开发时容易改错文件
- 回归测试范围膨胀

建议修复：

- 明确正式页清单
- 将历史页移到 `docs/archived` 或单独目录

#### 3.9 后台模块部分页面直接重复创建 PDO 连接，技术债较重

涉及页面：

- 多数 `admin/*.php`

问题说明：

- 有的页面用 `database_admin.php`
- 有的页面自己 new PDO

影响：

- 事务、异常、连接策略不统一

建议修复：

- 后台统一走一套数据库初始化入口

## 6. 结论

后台正式主模块目前可支撑日常使用，尤其是：

- 登录
- 任务创建与执行
- 文章管理
- 素材管理
- AI 配置
- 网站设置

但从“功能与配置完全一致”和“后台运维安全性”角度看，仍有少量治理项需要继续处理：

- 安全层面：AI 密钥仍为明文存储
- 维护层面：后台并行版本页面较多
- 技术债层面：数据库初始化入口尚未完全统一

整体评价：

- 可用性：8.5/10
- 业务完整性：8.5/10
- 配置一致性：8.5/10
- 安全性：7/10
