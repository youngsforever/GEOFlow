## 1. 项目概述

GEO+AI 智能内容生成系统是一个基于 PHP 的内容生产与发布平台，核心目标是把以下能力整合到同一套后台中：

- AI 模型配置与调用
- 标题库、关键词库、图片库、知识库管理
- 任务创建、调度、执行、恢复
- 文章草稿、审核、发布流程
- SEO 元数据输出与前台展示

系统当前已经从 SQLite 运行时架构切换到 PostgreSQL。  
SQLite 相关内容只保留给历史迁移和应急维护使用，不再属于主运行路径。

## 3. 当前技术架构

### 3.1 运行时组件

- `web`
  - PHP 8.3 CLI 内置服务器
  - 提供前台与后台页面
- `postgres`
  - 主数据库服务
  - 当前默认数据库名：`geo_system`
- `scheduler`
  - 周期执行 [`bin/cron.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/bin/cron.php)
  - 负责定时调度、清理、补任务
- `worker`
  - 常驻执行 [`bin/worker.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/bin/worker.php)
  - 负责领取队列任务并生成文章

### 3.2 数据库口径

当前主运行时数据库为 PostgreSQL。

运行时连接入口：

- [`includes/db_support.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/includes/db_support.php)
- [`includes/database.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/includes/database.php)
- [`includes/database_admin.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/includes/database_admin.php)

当前原则：

- 主应用运行时只允许 PostgreSQL
- SQLite 只保留给迁移脚本和维护脚本
- 运行时已不存在 SQLite schema 翻译层

### 3.3 队列与调度

当前任务链路：

1. 管理员在后台创建任务
2. `scheduler` 扫描可执行任务
3. 将任务写入 `job_queue`
4. `worker` 领取队列任务
5. AI 引擎生成文章
6. 生成结果写入文章表、运行记录表、心跳表

关键表：

- `tasks`
- `task_schedules`
- `job_queue`
- `task_runs`
- `worker_heartbeats`
- `articles`

## 6. 当前推荐运行方式

### 6.1 本地 Docker

推荐命令：

```bash
docker compose --profile scheduler up -d --build
```

说明：

- 会启动 `postgres`
- 会启动 `web`
- 会启动 `scheduler`
- 会启动 `worker`

### 6.2 本地 PHP 开发

如果不用 Docker，至少需要：

- 本地 PostgreSQL
- `pdo_pgsql`
- 正确的数据库环境变量

最小示例：

```bash
export DB_DRIVER=pgsql
export DB_HOST=127.0.0.1
export DB_PORT=5432
export DB_NAME=geo_system
export DB_USER=geo_user
export DB_PASSWORD=geo_password

php -S localhost:8080 router.php
```

## 7. 当前数据库维护方式

维护脚本：

- [`bin/db_maintenance.php`](/Users/laoyao/AI%20Coding/01-Projects/Active/GEO官网系统/bin/db_maintenance.php)

常用命令：

```bash
php bin/db_maintenance.php check
php bin/db_maintenance.php backup
php bin/db_maintenance.php checkpoint
```

在 PostgreSQL 模式下：

- `check`：检查连接与表数量
- `backup`：调用 `pg_dump`
- `checkpoint`：执行数据库 checkpoint
