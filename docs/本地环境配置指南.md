## 🔧 环境要求

### 必需软件
- **PHP**: >= 7.4 (推荐 8.0+)
- **PostgreSQL**: >= 14

### 必需的PHP扩展
- ✅ `pdo` - 数据库抽象层
- ✅ `pdo_pgsql` - PostgreSQL数据库支持
- ✅ `json` - JSON处理
- ✅ `mbstring` - 多字节字符串处理
- ✅ `session` - 会话管理
- ✅ `curl` - HTTP请求（AI API调用）

### 推荐配置
```ini
memory_limit = 256M
max_execution_time = 300
upload_max_filesize = 10M
post_max_size = 10M
```

## 📝 详细配置

### 1. 安装PHP（macOS）

```bash
# 使用Homebrew安装
brew install php

# 验证安装
php --version

# 检查扩展
php -m | grep -E "pdo|pgsql|json|mbstring|curl"
```

### 2. 安装PHP（Ubuntu/Debian）

```bash
# 更新包列表
sudo apt update

# 安装PHP及扩展
sudo apt install php php-cli php-pgsql php-mbstring php-json php-curl postgresql-client

# 验证安装
php --version
```

### 3. 安装PHP（Windows）

1. 下载PHP: https://windows.php.net/download/
2. 解压到 `C:\php`
3. 添加到系统PATH
4. 复制 `php.ini-development` 为 `php.ini`
5. 启用扩展（去掉分号）:
   ```ini
   extension=pdo_pgsql
   extension=mbstring
   extension=curl
   ```

### 4. 目录权限设置

```bash
# 确保关键目录可写
chmod -R 755 data/db
chmod -R 755 logs
chmod -R 755 uploads

# 如果需要，修改所有者
chown -R $(whoami) data logs uploads
```

### 5. 配置文件说明

#### `includes/config.php`
系统核心配置文件，包含：
- 网站基本信息
- 数据库连接配置
- 管理员账户
- 安全设置
- 上传配置

**重要配置项**:
```php
define('SITE_NAME', '智能GEO内容系统');
DB_DRIVER=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=geo_system
DB_USER=geo_user
DB_PASSWORD=geo_password
define('ADMIN_USERNAME', 'admin');
define('SECRET_KEY', 'your-secret-key-change-this-in-production');
```

⚠️ **生产环境必须修改 `SECRET_KEY`**

## 💻 常用命令

### 启动服务器

```bash
# 使用启动脚本（推荐）
./start.sh

# 手动启动
php -S localhost:8080 router.php

# 指定其他端口
php -S localhost:3000 router.php
```

### 停止服务器

```bash
# 在运行服务器的终端按
Ctrl + C
```

### 环境检查

```bash
# 运行环境检查脚本
./check-env.sh

# 手动检查PHP版本
php --version

# 检查PHP扩展
php -m

# 检查PHP配置
php --ini
```

### 查看日志

```bash
# 查看今日应用日志
tail -f logs/$(date +%Y-%m-%d).log

# 查看批量执行日志
tail -f logs/batch_*.log

# 查看任务管理器日志
tail -f logs/task_manager_$(date +%Y-%m-%d).log

# 查看所有日志
ls -lh logs/
```

### 数据库操作

```bash
# 检查数据库连接
php bin/db_maintenance.php check

# 连接PostgreSQL
psql -h 127.0.0.1 -U geo_user -d geo_system

# 查看所有表
\dt

# 查看表结构
\d articles
```

### 备份数据库

```bash
# 创建备份
cp data/db/blog.db data/db/blog_backup_$(date +%Y%m%d_%H%M%S).db

# 查看备份
ls -lh data/db/blog_backup_*.db
```

## 🔍 故障排除

### 问题1: 页面显示空白

**可能原因**:
- PHP错误
- 数据库连接失败
- 文件权限问题

**解决方法**:
```bash
# 1. 启用错误显示
php -S localhost:8080 router.php -d display_errors=1

# 2. 检查错误日志
tail -f logs/$(date +%Y-%m-%d).log

# 3. 检查数据库
ls -lh data/db/blog.db
```

### 问题2: 无法启动服务器

**可能原因**:
- 端口被占用
- PHP未安装
- 权限不足

**解决方法**:
```bash
# 1. 检查端口占用
lsof -i :8080

# 2. 使用其他端口
php -S localhost:3000 router.php

# 3. 检查PHP
which php
php --version
```

### 问题3: 数据库错误

**可能原因**:
- 数据库文件不存在
- 权限不足
- SQLite扩展未安装

**解决方法**:
```bash
# 1. 检查数据库文件
ls -lh data/db/blog.db

# 2. 检查权限
chmod 755 data/db
chmod 644 data/db/blog.db

# 3. 检查SQLite扩展
php -m | grep sqlite

# 4. 重新初始化
访问: http://localhost:8080/install.php
```

### 问题4: 无法登录后台

**可能原因**:
- 用户名密码错误
- Session配置问题
- 数据库问题

**解决方法**:
```bash
# 1. 重置管理员密码
php -r "echo password_hash('yaodashuai', PASSWORD_DEFAULT);"

# 2. 检查session目录
php -r "echo session_save_path();"

# 3. 清除session
rm -rf /tmp/sess_*
```

### 问题5: AI功能无法使用

**可能原因**:
- API密钥未配置
- 网络连接问题
- API限额用尽

**解决方法**:
1. 检查AI模型配置
2. 测试API连接
3. 查看错误日志
