### ✅ 修复 1.1：task_health_check.php - 添加 session_start()

**文件**: `admin/task_health_check.php`  
**问题**: 缺少 `session_start()`，导致 `$_SESSION` 为空，健康检查API总是返回"未登录"  
**修复**: 在第7行后添加 `session_start();`

**修改前**（第7-9行）:
```php
define('FEISHU_TREASURE', true);

require_once '../includes/config.php';
```

**修改后**（第7-10行）:
```php
define('FEISHU_TREASURE', true);
session_start();

require_once '../includes/config.php';
```

**验证方法**:
```bash

curl -b cookies.txt "http://localhost:8080/geo_admin/task_health_check.php?action=status"
```

### ✅ 修复 1.3：start_task_batch.php - 转义路径参数

**文件**: `admin/start_task_batch.php`  
**问题**: 使用 `exec()` 执行命令时未转义路径，当路径包含空格时会失败  
**修复**: 将 `exec()` 改为 `proc_open()` 并使用数组参数

**修改前**（第110行）:
```php
$command = "{$php_path} {$script_path} {$task_id} > {$log_file} 2>&1 &";
exec($command);
```

**修改后**（第105-125行，替换整个启动逻辑）:
```php
// 使用 proc_open 启动进程，避免路径转义问题
$descriptorspec = [
    0 => ['pipe', 'r'],  // stdin
    1 => ['file', $log_file, 'a'],  // stdout
    2 => ['file', $log_file, 'a']   // stderr
];

$process = proc_open(
    [PHP_BINARY, $script_path, (string)$task_id],
    $descriptorspec,
    $pipes,
    dirname(__DIR__)  // 工作目录
);

if (is_resource($process)) {
    // 关闭stdin
    if (isset($pipes[0])) {
        fclose($pipes[0]);
    }
    
    // 不等待进程结束，让它在后台运行
    proc_close($process);
    
    // 等待PID文件生成（最多2秒）
    $pid_file = dirname(__DIR__) . '/logs/batch_' . $task_id . '.pid';
    $wait_count = 0;
    while (!file_exists($pid_file) && $wait_count < 20) {
        usleep(100000);  // 100ms
        $wait_count++;
    }
    
    if (file_exists($pid_file)) {
        $pid = trim(file_get_contents($pid_file));
        write_log("批量执行进程已启动 - 任务ID: $task_id, PID: $pid", 'INFO');
    } else {
        write_log("批量执行进程启动成功但PID文件未生成 - 任务ID: $task_id", 'WARNING');
    }
    
    echo json_encode(['success' => true, 'message' => '批量执行已启动']);
} else {
    throw new Exception('无法启动批量执行进程');
}
```

**验证方法**:
```bash
# 1. 确保项目路径包含空格（当前路径已包含："AI Coding"）
# 2. 在任务列表页面点击"开始批量执行"按钮
# 3. 检查进程是否成功启动
ps aux | grep batch_execute_task.php
# 4. 检查日志文件是否正常生成
tail -f logs/batch_*.log
```
