# 登录问题修复说明

## 🐛 问题描述

**现象**: 在管理员登录页面输入用户名和密码后，跳转的后台页面打不开，需要刷新一下才能进入正式的管理员后台。

**错误信息**:
```
SQLSTATE[HY000]: General error: 1 no such column: last_login
```

---

## 🔍 问题分析

### 根本原因
在 `admin/index.php` 文件的第54行，登录成功后尝试更新管理员的 `last_login` 字段：

```php
// 更新最后登录时间
$stmt = $db->prepare("UPDATE admins SET last_login = CURRENT_TIMESTAMP WHERE id = ?");
$stmt->execute([$admin['id']]);
```

但是数据库中的 `admins` 表并没有 `last_login` 字段，导致SQL执行失败，抛出异常。

### 数据库表结构
当前 `admins` 表的字段：
```
id              INTEGER
username        VARCHAR(50)
password        VARCHAR(255)
display_name    VARCHAR(100)
email           VARCHAR(100)
bio             TEXT
avatar          VARCHAR(200)
website         VARCHAR(200)
created_at      DATETIME
```

**缺少**: `last_login` 字段

---

## ✅ 修复方案

### 已实施的修复
修改了 `admin/index.php` 文件，移除了更新 `last_login` 字段的代码，改为记录登录日志：

**修改前**（第48-59行）:
```php
if ($admin && password_verify($password, $admin['password'])) {
    $_SESSION['admin_logged_in'] = true;
    $_SESSION['admin_id'] = $admin['id'];
    $_SESSION['admin_username'] = $admin['username'];

    // 更新最后登录时间
    $stmt = $db->prepare("UPDATE admins SET last_login = CURRENT_TIMESTAMP WHERE id = ?");
    $stmt->execute([$admin['id']]);

    // 直接跳转到仪表盘
    header('Location: dashboard.php');
    exit;
}
```

**修改后**（第48-58行）:
```php
if ($admin && password_verify($password, $admin['password'])) {
    $_SESSION['admin_logged_in'] = true;
    $_SESSION['admin_id'] = $admin['id'];
    $_SESSION['admin_username'] = $admin['username'];

    // 记录登录日志
    write_log("管理员登录: {$admin['username']}", 'INFO');

    // 直接跳转到仪表盘
    header('Location: dashboard.php');
    exit;
}
```
